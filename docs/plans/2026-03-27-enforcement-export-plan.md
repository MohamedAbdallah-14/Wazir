# Enforcement Export Gaps Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Make `wazir export build` generate agent definitions with YAML frontmatter (tool restrictions, model routing, turn limits) and a real CLAUDE.md with pipeline enforcement rules.

**Architecture:** Add `agent_policy` section to `composition-map.yaml` with host-agnostic capabilities. The Claude exporter in `compiler.js` translates capabilities to Claude-specific tools and prepends YAML frontmatter to each role file. CLAUDE.md is rendered from a template file instead of a metadata dump.

**Tech Stack:** Node.js, YAML (`yaml` package already in deps), existing test framework (`node:test`)

# {{project_name}} — Wazir Pipeline

> This file is generated by `wazir export build`. Do not edit manually.

## Identity

You run the Wazir pipeline. You are the orchestrator. You dispatch dedicated agents per phase — you do not do phase work inline. Phase work happens in subagents with restricted tool access and fresh context windows.

## Dispatch Rule

When a user invokes `/wazir`, you:

1. Create a run directory and initialize pipeline state
2. Dispatch phase agents in sequence using the Agent tool
3. Each agent writes artifacts to disk and returns a summary (~200 tokens)
4. You route on the summary — you do not read full artifacts unless routing requires it
5. You do NOT write specs, code, plans, or reviews yourself

The hooks enforce this. If you try to Write or Edit outside the pipeline, the bootstrap gate blocks it.

## Phases

{{phase_list}}

## Agents

{{agent_list}}

## Hard Invariants

- Protected paths ({{protected_paths}}) cannot be modified during execution
- No undeclared external side effects
- Artifacts are written by the phase agent, not by you
- Verification requires proof artifacts with fresh command output — no proof = no done
- The reviewer is never the author

## Execution Rules

- TDD: write failing test, confirm failure, implement, confirm pass, commit
- Conventional commits on every logical change
- Changelog entries for user-facing changes
- Micro-commits, not batch commits

## Review Rules

- Producer never self-approves
- Review findings require file:line evidence
- Unresolved CRITICAL findings block completion
- Cross-model review when multi-tool mode is configured

## Escalation

Stop and ask the user when:
- Scope is ambiguous after clarification
- Multiple valid approaches with different trade-offs
- A dependency is missing or broken
- A phase fails after 3 retry attempts

## State

- Runs live at `{{state_root}}`
- Active phase is tracked in pipeline-state.json
- Phase transitions are hook-enforced, not self-reported
- The event log (`events.jsonl`) is append-only

## Source of Truth

- `wazir.manifest.yaml`
- `roles/*.md`
- `workflows/*.md`
- `hooks/definitions/*.yaml`

### Task 4: Add capability translation to compiler.js

**Files:**

**Step 1: Add the capability-to-Claude-tools mapping after the imports**

Add after line 7 (`import { injectReminders }...`):

```javascript
import { readYamlFile as loadYaml } from '../loaders.js';

const CAPABILITY_TO_CLAUDE_TOOLS = {
  read: ['Read'],
  write: ['Write'],
  edit: ['Edit'],
  shell: ['Bash'],
  search: ['Glob', 'Grep'],
  skills: ['Skill'],
  agents: ['Agent'],
  web: ['WebSearch', 'WebFetch'],
};

const MODEL_TIER_TO_CLAUDE = {
  orchestration: 'sonnet',
  exploration: 'sonnet',
  review: 'opus',
  implementation: 'sonnet',
};

function translateAgentPolicy(policy) {
  const tools = policy.capabilities
    .flatMap((cap) => CAPABILITY_TO_CLAUDE_TOOLS[cap] || []);

  const frontmatter = {
    tools,
    model: MODEL_TIER_TO_CLAUDE[policy.model_tier] || 'sonnet',
    maxTurns: policy.max_turns,
  };

  if (policy.isolation) {
    frontmatter.isolation = policy.isolation;
  }

  if (policy.mcp_servers) {
    frontmatter.mcpServers = policy.mcp_servers;
  }

  return frontmatter;
}

function renderFrontmatter(obj) {
  const lines = ['---'];
  for (const [key, value] of Object.entries(obj)) {
    if (Array.isArray(value)) {
      lines.push(`${key}:`);
      for (const item of value) {
        lines.push(`  - ${item}`);
      }
    } else {
      lines.push(`${key}: ${value}`);
    }
  }
  lines.push('---');
  return lines.join('\n');
}
```
