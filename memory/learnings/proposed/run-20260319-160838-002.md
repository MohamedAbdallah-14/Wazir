# Proposed Learning: Never use interactive CLI prompts in agent environments

## Scope
- **Roles:** executor
- **Stacks:** node
- **Concerns:** tooling, cli

## Evidence
- `@inquirer/prompts` select() calls ALWAYS fail in Claude Code's Bash tool (non-TTY)
- The init command was unusable from within the pipeline
- Zero-config auto-init with `--interactive` opt-in fixed this

## Learning
CLI commands invoked from agent environments must never use interactive prompts by default. Default to auto-detection/inference. Provide `--interactive` flag for TTY environments only.

## Expected Benefit
CLI commands work reliably in both human and agent contexts without TTY failures.

## Confidence
- **Level:** high
- **Basis:** multi-run recurrence (init always fails in Claude Code)
