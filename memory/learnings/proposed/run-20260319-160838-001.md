# Proposed Learning: Always enforce scope coverage gate

## Scope
- **Roles:** clarifier, planner
- **Stacks:** all
- **Concerns:** process, quality

## Evidence
- Run run-20260319-133426: 21 input items were autonomously reduced to 5 tasks (Item 19 description)
- The clarifier had no hard gate preventing scope reduction
- User had to manually intervene and create a second input file with remaining items

## Learning
Before presenting an execution plan, count items in input vs tasks in plan. If `tasks_in_plan < items_in_input`, STOP and require user approval. Never autonomously tier, defer, or drop items.

## Expected Benefit
Prevents silent scope reduction where the agent delivers less than what was requested.

## Confidence
- **Level:** high
- **Basis:** user correction after real incident
