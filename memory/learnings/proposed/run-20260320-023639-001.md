# ESM static imports are hoisted above top-level code

When suppressing Node.js warnings in an ESM entrypoint, `process.emit` overrides placed before `import` statements still run AFTER imports because ESM hoists all static imports. Use dynamic `import()` for modules that trigger the warning, so the override is active when the module loads.

**Recurrence:** First time, but affects any ESM CLI entrypoint.
